# Documentation Index

This directory contains detailed documentation files for the Universal AI Autofill Assistant, covering system setup, user workflows, architectural specifications, and internal API designs.

---

## 📂 Documentation Files

1. **[Project Documentation](file:///docs/Project_Documentation.md):**
   * Overview of the project scope, problem statement, objectives, and functional modules.
2. **[Installation & Setup Guide](file:///docs/Installation_Guide.md):**
   * Developer environment setup instructions, including Android Studio config, physical device setup, and deploying APKs using ADB.
3. **[User Manual](file:///docs/User_Manual.md):**
   * Guide for end-users, explaining onboarding, custom profiles, OCR document scanning, floating bubble overlay, and text expansion shortcuts.
4. **[Technical & Architectural Documentation](file:///docs/Technical_Documentation.md):**
   * Deep dive into technical details: screen scraping algorithms, field-matching priority workflows, OCR parsers, and the encryption/security model.
5. **[System Architecture (Visual Flows)](file:///docs/System_Architecture.md):**
   * Mermaid sequence and block diagrams mapping user navigation flows, accessibility events, and camera capture processes.
6. **[Internal API & Method Reference](file:///docs/API_Documentation.md):**
   * Programmatic specifications for key methods, ViewModels, and services (such as `PinManager` and `SmartAccessibilityService`).
7. **[Diagram Assets Reference](file:///docs/diagrams/README.md):**
   * Guide on how system diagrams are generated and updated.

## 🎓 Academic Presentation Tips
If you are presenting this repository for academic evaluation:
* Walk evaluators through the **[Project Documentation](file:///docs/Project_Documentation.md)** first to explain the project scope.
* Present the **[System Architecture](file:///docs/System_Architecture.md)** flowcharts to demonstrate your understanding of the app's lifecycle.
* Refer to the **[Technical Documentation](file:///docs/Technical_Documentation.md)** when explaining the screen traversal algorithm and security features.
